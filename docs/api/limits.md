# DataWorks OpenAPI 使用限制与计费

## 版本要求

各功能模块的 OpenAPI 对 DataWorks 的版本要求与功能本身对 DataWorks 版本的要求一致。

## 使用限制

### QPS 限制与每日调用次数

不同 DataWorks 版本中的读、写 OpenAPI 分别有相应的 QPS 限制和每日调用次数限制：

| 版本 | QPS 限制 | 每日调用次数上限 |
|------|----------|------------------|
| 基础版 | 所有 API 总 QPS ≤ 5 | 每日 API 总调用次数 ≤ 100 次 |
| 标准版 | - | 每日 API 总调用次数 ≤ 1,000 次 |
| 专业版 | - | 每日 API 总调用次数 ≤ 10,000 次 |
| 企业版 | 读 API 总 QPS ≤ 50<br>写 API 总 QPS ≤ 20 | 每日 API 总调用次数不限 |

### 重要说明

- **统计口径**：每个主账号及其子账号在每个地域的累计值
- **超限处理**：超过 QPS 或每日调用次数限制后，将会出现调用失败
- **调用计数**：API 调用成功和调用失败的次数都会计入每日调用限制次数中
- **读写类型判断**：可根据 API 名称判断，例如 `CreateFile` 为**写** API，`ListFiles` 为**读** API

## 免费额度与计费方式

各版本的 OpenAPI 分别提供相应的每月免费调用额度：

| 版本 | 免费调用额度 | 超出免费调用额度后按量计费 |
|------|--------------|----------------------------|
| 基础版 | 3,100 次/月 | 有调用限制，不支持额外按量付费 |
| 标准版 | 31,000 次/月 | 有调用限制，不支持额外按量付费 |
| 专业版 | 310,000 次/月 | 有调用限制，不支持额外按量付费 |
| 企业版 | 1,000,000 次/月 | **0.3 元/万次** |

### 计费说明

- **免费额度周期**：每个自然月，每月 1 日重置
- **统计范围**：每个主账号（包含子账号）在每个地域的 OpenAPI
- **实际可调用量**：受日调用次数限制，基础版、标准版、专业版的每月实际可调用量可能无法超过免费调用额度
- **按量计费**：仅企业版支持免费额度用尽后按量计费
- **特殊环境计费**：
  - 政务云：**0.6 元/万次**
  - 金融云：**0.57 元/万次**
  - 实际计费价格以最终账单显示为准

### 计费规则

- 只有 API **调用成功**后，才会消耗免费调用额度并计量计费
- API 调用**失败**不会消耗免费调用额度且不计量计费

## 调用建议

1. **合理控制并发**：避免触发 QPS 限制
2. **批量操作**：尽量使用批量 API 减少调用次数
3. **错误处理**：做好错误重试和限流处理
4. **监控使用情况**：通过开放平台查看 API 使用情况

## 查看 OpenAPI 使用情况

您可以通过开放平台查看：
- OpenAPI 免费额度使用情况
- OpenAPI 调用次数
- OpenAPI 调用明细

访问路径：登录 DataWorks 控制台 > 开放平台 > 开发者后台
