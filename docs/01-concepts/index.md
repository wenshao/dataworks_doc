# DataWorks 核心概念

## 概念地图

```
┌─────────────────────────────────────────────────────────┐
│                    DataWorks                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────────┐      ┌─────────────┐      ┌─────────┐ │
│  │ 工作空间    │ ──▶ │   节点      │ ──▶ │  实例   │ │
│  │ Workspace   │      │   Node      │      │ Instance│ │
│  └─────────────┘      └─────────────┘      └─────────┘ │
│         │                     │                        │
│         ▼                     ▼                        │
│  ┌─────────────┐      ┌─────────────┐                  │
│  │   数据源    │      │   工作流    │                  │

└─────────────────────────────────────────────────────────┘
```

## 核心概念

### 工作空间 (Workspace)

DataWorks 的基本组织单元，一个工作空间对应一个计算引擎项目。

**类比：** 一个"项目"或"工程"

**关键属性：**
- `project_identifier`：工作空间ID（全局唯一）
- `name`：工作空间名称
- `status`：状态（NORMAL/FROZEN等）
- 关联的计算引擎（MaxCompute/EMR/Hologres）

**建议：**
- 开发、测试、生产使用不同工作空间
- 一个部门或项目对应一个工作空间

### 节点 (Node)

数据开发的最小执行单元，代表一个具体的计算任务。

**节点类型：**

| 类型 | file_type | 说明 |
|------|-----------|------|
| ODPS SQL | 10 | MaxCompute SQL 任务 |
| Shell | 21 | Shell 脚本任务 |
| Spark | 24 | Spark 计算任务 |
| 离线同步 | 27 | 数据同步任务 |
| 虚拟节点 | 35 | 不执行计算，用于依赖控制 |
| Python | 36 | Python 脚本任务 |

**生命周期：** 开发 → 提交 → 发布 → 运行

### 工作流 (Workflow)

节点的逻辑组织方式，将相关节点组织在一起。

**作用：**
- 组织有依赖关系的节点
- 整体发布和管理
- 可视化展示数据流

**类比：** 一个"有向无环图"(DAG)

### 实例 (Instance)

节点的一次具体执行记录。

**状态流转：**

```
WAIT_WAITING → WAIT_RUNNING → RUNNING → SUCCESS / FAILED
     ↓              ↓
  (等待时间)    (等待依赖)
```

**状态说明：**

| 状态 | 说明 |
|------|------|
| `WAIT_WAITING` | 等待调度时间到达 |
| `WAIT_RUNNING` | 等待上游节点完成 |
| `RUNNING` | 正在运行 |
| `SUCCESS` | 运行成功 |
| `FAILED` | 运行失败 |
| `CANCELLED` | 已取消 |

### 数据源 (DataSource)

DataWorks 与外部数据系统连接的配置。

**支持的数据源类型：**

| 类别 | 数据源 |
|------|--------|
| 关系型数据库 | MySQL, Oracle, SQL Server, PostgreSQL |
| 大数据 | MaxCompute, Hive, Kudu, Hologres |
| NoSQL | MongoDB, Redis, HBase |
| 消息队列 | Kafka, RocketMQ |
| 文件 | OSS, HDFS, FTP, S3 |

### 文件 (File)

节点的存储形式，包含节点的代码和配置。

**文件类型与节点类型的对应关系：**

| 节点类型 | 文件类型 |
|----------|----------|
| SQL 节点 | 代码文件 (.sql) |
| Shell 节点 | 代码文件 (.sh) |
| Python 节点 | 代码文件 (.py) |
| Spark 节点 | 代码文件 + 资源文件 (.jar) |

### 调度参数 (Schedule Parameters)

在任务运行时动态替换的变量。

**常用参数：**

| 参数 | 说明 | 示例值 |
|------|------|--------|
| `${bizdate}` | 业务日期 | 20250101 |
| `${cyctime}` | 调度时间 | 2025-01-01 00:00:00 |
| `${node_id}` | 节点ID | 节点的唯一标识 |

**业务日期 vs 运行日期：**

| 场景 | 运行日期 | 业务日期 | 说明 |
|------|----------|----------|------|
| T+1 日处理 | 2025-01-02 | 20250101 | 处理昨天的数据 |
| 当日处理 | 2025-01-01 | 20250101 | 处理今天的数据 |

## 数仓分层概念

```
┌────────────────────────────────────────────────┐
│                    ADS                         │  应用数据层
│              (Application Data Store)          │  面向具体应用
├────────────────────────────────────────────────┤
│                    DWS                         │  汇总数据层
│            (Data Warehouse Summary)            │  主题汇总
├────────────────────────────────────────────────┤
│                    DWD                         │  明细数据层
│          (Data Warehouse Detail)               │  明细事实
├────────────────────────────────────────────────┤
│                    ODS                         │  原始数据层
│           (Operational Data Store)             │  原始同步
└────────────────────────────────────────────────┘
```

### ODS 层

- **用途**：原始数据缓冲区
- **特点**：与源系统数据一致
- **命名规范**：`ods_{源系统}_{表名}`

### DWD 层

- **用途**：明细数据层
- **特点**：对 ODS 数据进行清洗、规范化
- **命名规范**：`dwd_{业务域}_{事实表}`

### DWS 层

- **用途**：汇总数据层
- **特点**：按主题域进行轻度汇总
- **命名规范**：`dws_{业务域}_{汇总粒度}`

### ADS 层

- **用途**：应用数据层
- **特点**：面向具体应用需求
- **命名规范**：`ads_{业务}_{用途}`
