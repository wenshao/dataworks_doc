# DataWorks 基本概念

## 核心概念

### 工作空间 (Workspace)

工作空间是 DataWorks 的基本组织单元，一个工作空间对应一个 MaxCompute 项目（或 EMR 集群）。

**核心属性：**
- 工作空间名称
- 关联的计算引擎（MaxCompute/EMR/Hologres）
- 所在地域
- 成员权限配置

**建议：**
- 一个部门或一个项目对应一个工作空间
- 开发、测试、生产环境使用不同工作空间

### 业务流程 (Business Flow)

业务流程是节点的逻辑组织方式，用于将相关节点组织在一起。

**特点：**
- 一个工作流包含多个节点
- 节点间可以配置依赖关系
- 工作流可整体提交发布

### 节点 (Node)

节点是数据开发的最小执行单元，对应一个具体的计算任务。

**节点类型：**
| 类型 | 说明 |
|------|------|
| ODPS_SQL | MaxCompute SQL 节点 |
| SHELL | Shell 脚本节点 |
| SPARK | Spark 计算节点 |
| MAPREDUCE | MapReduce 计算节点 |
| PYTHON | Python 脚本节点 |
| SYNC | 数据同步节点 |
| VIRTUAL | 虚拟节点（不执行实际计算） |

### 文件 (File)

文件是节点的存储形式，包含节点的代码和配置信息。

**文件类型：**
- 代码文件：SQL、Shell、Python 等
- 资源文件：JAR、PY 等
- 配置文件：调度配置等

### 任务实例 (Instance)

实例是节点的一次具体执行记录。

**实例状态：**
| 状态 | 说明 |
|------|------|
| WAIT_WAITING | 等待调度时间 |
| WAIT_RUNNING | 等待上游完成 |
| RUNNING | 正在运行 |
| SUCCESS | 运行成功 |
| FAILED | 运行失败 |
| CANCELLED | 已取消 |

### 调度参数

调度参数是在任务运行时动态替换的变量。

**常用参数：**
| 参数 | 说明 | 示例值 |
|------|------|--------|
| ${bizdate} | 业务日期 | 20250101 |
| ${cyctime} | 调度时间 | 2025-01-01 00:00:00 |
| ${dt} | 分区值 | 2025-01-01 |

### 资源组

资源组是运行任务的计算资源集合。

**资源组类型：**
- 默认资源组：共享资源
- 独享资源组：独享资源
- Serverless 资源组：按需使用

### 数据源

数据源是 DataWorks 与外部数据系统连接的配置。

**支持的数据源：**
- 关系型数据库：MySQL、Oracle、SQL Server 等
- 大数据引擎：MaxCompute、Hive、Kudu 等
- NoSQL：MongoDB、Redis、HBase 等
- 消息队列：Kafka、RocketMQ 等
- 文件存储：OSS、HDFS、FTP 等

### 数据目录

数据目录是统一的数据资产元数据管理系统。

**核心功能：**
- 数据表检索
- 数据血缘分析
- 数据权限查看
- 数据资产展示

## 数据分层

### ODS 层 (Operational Data Store)

- 描述：原始数据层
- 特点：与源系统数据一致
- 作用：数据缓冲区

### DWD 层 (Data Warehouse Detail)

- 描述：明细数据层
- 特点：对 ODS 数据进行清洗、规范化
- 作用：统一数据标准

### DWS 层 (Data Warehouse Summary)

- 描述：汇总数据层
- 特点：按主题域进行轻度汇总
- 作用：提供汇总指标

### ADS 层 (Application Data Store)

- 描述：应用数据层
- 特点：面向具体应用需求
- 作用：支撑业务应用

## 数仓建模概念

### 维度表

描述业务实体的属性表，如用户表、商品表。

**特点：**
- 变化频率较低
- 包含丰富的描述性属性
- 作为分析的角度

### 事实表

记录业务过程的表，如订单表、日志表。

**类型：**
- 事务事实表：记录业务事务
- 周期快照事实表：定期记录状态
- 累积快照事实表：记录过程全生命周期

### 维度

观察数据的角度，如时间维度、地域维度。

### 指标

度量的数值化表达，如销售额、用户数。

**指标类型：**
- 原子指标：不可再拆分的指标
- 派生指标：基于原子指标计算得出
- 衍生指标：基于派生指标进一步计算

## 开发流程概念

### 开发环境

用于代码开发和测试的环境。

### 生产环境

用于正式运行任务的环境。

### 发布中心

用于将开发环境的代码发布到生产环境的功能模块。

### 补数据

对历史数据重新计算的过程。

**补数据类型：**
- 指定业务日期补数据
- 指定时间范围补数据
- 自定义补数据

## 数据治理概念

### 数据质量

衡量数据好坏的标准，包括准确性、完整性、一致性等。

### 数据标准

数据定义、格式、取值等的统一规范。

### 数据血缘

数据之间的来源和去向关系。

### 数据资产

具有价值、可被使用的数据资源。
